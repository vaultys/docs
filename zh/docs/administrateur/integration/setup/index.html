管理员对员工的普查​

管理员对所有员工进行全面普查。每个员工可以通过以下方式进行识别：

  • 姓
  • 名
  • 专业电子邮件
  • 专业电话（可选）
  • 组（可选，用逗号分隔）。组对于管理访问权限非常有用。通常可以想象到一些组，比如营销，会计，销售，管理，人力资源等等，这些组将共享相同的资源。
提示

组的概念非常灵活。对于一些代表客户运营的通信公司等活动，可以想象为每个客户创建一个组，以便与有限的员工列表共享LinkedIn、Facebook和其他社交网络的连接。

应用程序和需要保护的网络服务清单​

在实施任何措施之前，管理员会仔细识别需要通过SmartLink进行保护的应用程序和网络服务。这一战略性步骤确保只有关键和敏感资源被整合到系统中。通常，员工会使用他们在浏览器中保存的书签和/或密码管理器。他们可以向管理员提供使用的应用程序的以下信息：

  • 应用程序名称
  • URL，例如 https://application.en.ligne.com)
  • 是否使用密码进行登录。如果是，密码是否在组内共享，还是个人使用。如适用，命名组

完成清单后，整理后的文件可以直接集成到SmartLink中

提示

不要过分关注细节，特别是分享或组的细节，可以随时在SmartLink中直接调整这些设置。

创建管理员账户​

在确定应用程序并对员工进行普查后，管理员将创建SmartLink的管理员账户。这个集中的账户将有效管理企业内的安全策略和访问权限。

通知员工​

员工会及时被告知SmartLink的整合以及由此带来的对应用程序和服务访问的变化。清晰详细的沟通对于确保平稳过渡和最小化日常运营中的任何干扰至关重要。

可用资源列表​

最后，会建立一份完整的通过SmartLink可用的资源列表，并提供给员工。这一资源有助于快速访问关键的安全应用程序信息，并加强企业内的透明度。

通过遵循这一系统方法，企业可以有效地整合SmartLink，从而增强数据安全性，优化访问和用户管理。
